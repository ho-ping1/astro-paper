Mijn eerste week bestond uit een introductie over wat ik allemaal kon verwachten tijdens mijn stage. De eerste dag bestond grotendeels uit een uitleg over regels op het bedrijf en ook welke applicaties ik allemaal zou gebruiken. Verder had ik mijn eerste zicht kunnen nemen op de backend code waarin ik zou werken. Ik had al mijn eerste opstakels met het opzetten van de werkomgeving. Ze gebruiken custom terminal commando's voor het binnenhalen van projecten waarvoor ik hulp nodig had om te weten hoe het allemaal werkten. Er waren ook veel errors tijdens het installeren van packages waardoor ik pas woensdag kon beginnen met mijn eerste lijnen coden te schrijven.

De rest van de week heb ik mijn tijd gespendeerd met het bestuderen van de code en ben ik ook begonnen met mijn eerste controller te schrijven voor de api. Mijn eerste code compile error dat ik kreeg was een transaction error waarbij er meerdere transactions bezig waren wanneer ik een api call probeerden te testen via Postman. Ik heb dit dan moeten oplossen via een boolean parameter in een if-case om de foutmelding te vermijden.

Terugkijkend op deze week heb ik veel bijgeleerd over de workflow een de basis van de codebase. Ondanks de hindernissen voelde ik me zelfverzekerder in de werkomgeving en heb ik het gevoel dat ik een goede basis heb gelegd voor de komende weken.
